<decisions>
## Implementation Decisions

### Report content & structure
- Summary level detail per step: step name, outcome (completed/failed/skipped), user inputs, and branching decisions taken
- No full audit trail in PDF — keep it concise and readable
- Form data shown as field labels + values entered (data only, no visual form snapshots)
- Branching decisions show path taken only — no listing of alternatives
- Nested child workflows (Workflow Proxy) inlined in the parent report, indented or grouped under the proxy step

### Report appearance
- Formal/professional tone: cover page with workflow name, execution date, and outcome summary
- Structured headers and table formatting suitable for auditors or clients
- Minimal branding: small "Generated by BrainPal" in footer only — no logo or prominent branding
- Portrait page orientation
- Subtle color accents: mostly black and white with occasional color highlights for key outcomes (pass/fail)

### Export trigger & delivery
- Export button on the history detail screen only (not on execution screen)
- Mobile: generate PDF then open native share sheet (save to Files, email, AirDrop, etc.)
- Web: Claude's discretion on auto-download vs preview in new tab
- Filename convention: `{WorkflowName}_run-{runId}.pdf`

### Progress & confirmation patterns
- ZIP import: inline progress indicator near the import button (not full-screen overlay) — user can still see the screen
- Confirmation dialogs required for: ABORT and delete only (STOP is reversible, no confirmation needed)
- Confirmation dialog style: native alert dialog with Cancel/Confirm buttons — no custom modal
- PDF generation: disable export button during generation (no visible progress indicator)

### Claude's Discretion
- Web PDF delivery mechanism (auto-download vs new tab preview)
- PDF generation library choice
- Exact cover page layout and typography
- Pagination strategy for large execution histories
- Zustand selector optimization approach
- Loading skeleton/spinner design for paginated history

</decisions>
