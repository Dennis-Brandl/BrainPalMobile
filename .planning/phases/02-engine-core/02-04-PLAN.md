    **Create PackageImporter class** (`src/import/package-importer.ts`):
    - Constructor takes: IMasterWorkflowRepository, IMasterEnvironmentRepository, IMasterActionRepository, IImageRepository, IExecutionLogger (for logging PACKAGE_IMPORTED events)

    - `importPackage(zipData: Uint8Array)`: Promise<ImportResult>
      1. Call extractPackage(zipData) to get ExtractedPackage
      2. Validate the extracted content (all JSON parses succeeded, manifest is valid)
      3. For each workflow in extracted.workflows:
         a. Check if master workflow with same OID already exists (IMasterWorkflowRepository.getByOid)
         b. If exists: **version replacement** -- delete old spec and all associated environments, actions, and images for that OID (cascade). Then save new.
         c. If not exists: save new master workflow
      4. For each environment: save via IMasterEnvironmentRepository.save()
      5. For each action: save via IMasterActionRepository.save()
      6. For each image: save via IImageRepository.save()
      7. Log PACKAGE_IMPORTED event
      8. Return ImportResult with success=true and list of imported workflow OIDs

    - Error handling:
      - If extractPackage throws PackageValidationError: return { success: false, workflowOids: [], error: errorMessage }
      - If any storage operation fails: this is an unexpected error, let it propagate (the caller handles)
      - The entire operation is all-or-nothing: validate everything in memory FIRST, then write to storage. If validation fails at any point, nothing is written. (Pitfall 4 prevention)

    - `deletePackage(workflowOid: string, activeWorkflowInstanceIds?: string[])`: Promise<void>
      - If activeWorkflowInstanceIds provided and non-empty: throw error indicating active instances exist (the UI layer will have already warned the user and decided to proceed or not -- if we get here with active IDs, it means proceed with cascade)
      - Delete master workflow (cascades to environments, actions, images via repository)
      - Does NOT delete runtime workflow instances -- those run from deep-copied data and can continue independently

    - Test all-or-nothing: if one workflow in a library package has invalid JSON, NONE of the workflows are stored
