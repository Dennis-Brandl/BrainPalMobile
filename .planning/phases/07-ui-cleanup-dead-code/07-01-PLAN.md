    **execute.tsx** -- Complete rewrite. Replace the placeholder stub with a smart routing screen:

    1. Import `useExecutionStore` from `../../src/stores/execution-store` and `useRouter` + `Href` from `expo-router`. Import `useFocusEffect` from `expo-router` to trigger redirect on tab focus (not just mount). Import `FontAwesome` from `@expo/vector-icons/FontAwesome` for the empty state icon.

    2. Read `activeWorkflows` from `useExecutionStore((s) => s.activeWorkflows)`. Derive `activeList` in a useMemo:
       ```ts
       const activeList = useMemo(() =>
         Object.values(activeWorkflows)
           .sort((a, b) => {
             // Most recently started workflow first (locked decision: "most recently started")
             const aTime = a.lastActivityAt ?? a.startedAt ?? '';
             const bTime = b.lastActivityAt ?? b.startedAt ?? '';
             return bTime.localeCompare(aTime);  // descending
           }),
         [activeWorkflows]
       );
       ```
       This sort is REQUIRED by locked decision: "auto-navigate to the most recently started active workflow." Without it, Object.values returns insertion-order which is non-deterministic for the user's intent. The ActiveWorkflowState interface has both `startedAt: string | null` and `lastActivityAt: string | null` fields available.

    3. Three-branch conditional rendering:
       - **Zero active workflows:** Show an empty state screen. Centered content with a FontAwesome `play-circle` icon (size 64, color `colors.textSecondary` with reduced opacity), heading "No Active Workflow", body text "Start a workflow from the Library tab on the Home screen". Use SafeAreaView with `edges={['bottom']}`, consistent styling with the Home screen empty states (same typography, spacing, colors from `@brainpal/ui`).
       - **Exactly one active workflow:** Use `useFocusEffect` to call `router.replace(`/execution/${activeList[0].instanceId}` as Href)` when the tab comes into focus and activeList.length === 1. Because activeList is sorted by lastActivityAt descending, `activeList[0]` is always the most recently started workflow. Show a brief loading/redirecting state (ActivityIndicator centered) while the redirect happens, so the screen is not blank.
       - **Multiple active workflows:** Render a FlatList of active workflows (using the already-sorted activeList). Each item shows the workflow name and a StateBadge (import from `../../src/components/workflow/WorkflowCard` -- check if StateBadge is exported separately, if not, create a simple inline badge). Tapping an item calls `router.push(`/execution/${item.instanceId}` as Href)`. Use the same card styling pattern as the Home screen's ActiveWorkflowCard. Include a header "Active Workflows" above the list.

    Important: Use `router.replace` (not `router.push`) for the single-workflow redirect so the user can press back to go to the previous tab, not back to a blank execute screen.
