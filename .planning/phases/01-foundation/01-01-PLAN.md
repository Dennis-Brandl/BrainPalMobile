    **Step 1: Create the Expo app as the starting point.**

    Run `npx create-expo-app@latest brainpal-mobile --template tabs` in a temporary location, then restructure into the monorepo layout. Alternatively, initialize the monorepo manually:

    1. Create the root `package.json` with npm workspaces:
       ```json
       {
         "name": "brainpal-mobile",
         "private": true,
         "workspaces": ["apps/*", "packages/*"],
         "overrides": {
           "react": "$react",
           "react-native": "$react-native"
         },
         "devDependencies": {
           "turbo": "~2.8.0",
           "typescript": "~5.8.0"
         }
       }
       ```

    2. Create `apps/mobile/` as an Expo app. The `apps/mobile/package.json` should:
       - Set `"name": "brainpal-mobile-app"` (the app, not the workspace root)
       - List Expo SDK 54, React 19.1.x, React Native 0.81.x as dependencies
       - List `expo-router ~6.x`, `expo-sqlite ~16.x`, `react-native-safe-area-context`, `react-native-gesture-handler`, `react-native-reanimated`, `react-native-screens`, `react-native-web`, `react-dom`, `@expo/metro-runtime`, `@expo/vector-icons` as dependencies
       - List `zustand ~5.x` as a dependency
       - List all four `@brainpal/*` packages as workspace dependencies (`"@brainpal/engine": "*"`, etc.)
       - Use `npx expo install` for Expo packages to get version-pinned dependencies

    3. Create `apps/mobile/app.json` with Expo configuration:
       - `expo.name`: "BrainPal Mobile"
       - `expo.slug`: "brainpal-mobile"
       - `expo.scheme`: "brainpal-mobile"
       - `expo.web.bundler`: "metro"
       - `expo.web.output`: "single"
       - `expo.plugins`: ["expo-router", "expo-sqlite"]
       - `expo.newArchEnabled`: true

    4. Create `apps/mobile/metro.config.js` with:
       - `getDefaultConfig` from `expo/metro-config`
       - WASM asset extension: `config.resolver.assetExts.push('wasm')`
       - COOP/COEP middleware for SharedArrayBuffer (use `credentialless` for COEP, not `require-corp`):
         ```javascript
         const { getDefaultConfig } = require('expo/metro-config');
         const config = getDefaultConfig(__dirname);
         config.resolver.assetExts.push('wasm');
         const originalEnhanceMiddleware = config.server.enhanceMiddleware;
         config.server.enhanceMiddleware = (middleware, metroServer) => {
           if (originalEnhanceMiddleware) {
             middleware = originalEnhanceMiddleware(middleware, metroServer);
           }
           return (req, res, next) => {
             res.setHeader('Cross-Origin-Embedder-Policy', 'credentialless');
             res.setHeader('Cross-Origin-Opener-Policy', 'same-origin');
             return middleware(req, res, next);
           };
         };
         module.exports = config;
         ```

    5. Create `apps/mobile/tsconfig.json` extending `../../tsconfig.base.json` with expo-router path aliases.

    **Step 2: Create the four shared packages.**

    Each package gets `package.json`, `tsconfig.json`, and `src/index.ts`:

    - **@brainpal/engine**: Pure TypeScript. No dependencies in Phase 1. `main` and `types` point to `src/index.ts`. Export a placeholder: `export const ENGINE_VERSION = '0.0.1';`
    - **@brainpal/protocol**: Pure TypeScript. No dependencies in Phase 1. Export placeholder: `export const PROTOCOL_VERSION = '0.0.1';`
    - **@brainpal/storage**: Depends on `expo-sqlite ~16.x`. Peer dependency on `react`. Export placeholder: `export const STORAGE_VERSION = '0.0.1';`
    - **@brainpal/ui**: Peer dependencies on `react` and `react-native`. Contains theme tokens. Exports theme from `src/theme/index.ts`.

    Package `tsconfig.json` files extend `../../tsconfig.base.json` with `composite: true`.
