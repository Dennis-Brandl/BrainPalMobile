**1. Create useCanvasScale hook** (`apps/mobile/src/hooks/useCanvasScale.ts`):

Calculates the uniform scale factor to fit a canvas of known dimensions into the available screen width.

```typescript
interface CanvasScaleResult {
  scale: number;
  scaledWidth: number;
  scaledHeight: number;
  containerWidth: number;
}

function useCanvasScale(canvasWidth: number, canvasHeight: number, padding?: number): CanvasScaleResult
```

Implementation:
- Use `useWindowDimensions()` to get screen width
- Subtract horizontal padding (default 2 * spacing.base = 32px) from screen width to get container width
- Scale = containerWidth / canvasWidth (always <= 1, never upscale)
- If canvasWidth <= containerWidth, scale = 1 (no scaling needed)
- scaledWidth = canvasWidth * scale
- scaledHeight = canvasHeight * scale

This is the "uniform scale-to-fit" approach from the CONTEXT decisions. Simple and correct.

**2. Create FormCanvas component** (`apps/mobile/src/components/form/FormCanvas.tsx`):

The scaled container that positions all form elements using absolute layout.

Props:
```typescript
interface FormCanvasProps {
  layout: FormLayoutEntry;           // from step's form_layout_config
  formData: Record<string, string>;  // current form values
  onFormDataChange: (key: string, value: string) => void;  // form input handler
  renderActionButtons?: () => React.ReactNode;  // slot for action buttons below canvas
  images?: Map<string, string>;      // filename -> base64 data URI for image elements
}
```

Implementation:
1. Call `useCanvasScale(layout.canvasWidth, layout.canvasHeight)` to get scale factor
2. Outer ScrollView (vertical scroll to accommodate tall forms that exceed screen height after scaling)
3. Inside ScrollView: a GestureDetector wrapping the canvas for pinch-to-zoom
4. Canvas container View with:
   - `width: layout.canvasWidth` (original canvas width)
   - `height: layout.canvasHeight` (original canvas height)
   - `transform: [{ scale }]` where scale is from the hook (or animated scale from pinch gesture)
   - `transformOrigin: 'top left'` (scale from top-left corner)
5. Canvas outer wrapper View with:
   - `width: scaledWidth` (actual space this takes on screen)
   - `height: scaledHeight` (actual height after scaling)
   - `overflow: 'hidden'` (clip anything outside canvas bounds)
6. For each element in `layout.elements`, render `<FormElementRenderer>` with absolute positioning:
   - `position: 'absolute'`
   - `left: element.x`
   - `top: element.y`
   - `width: element.width`
   - `height: element.height`
7. After the canvas, render `renderActionButtons?.()` for step completion buttons

**Pinch-to-zoom implementation** (from Research Pattern 5):
- Use `react-native-gesture-handler` Gesture.Pinch() with `react-native-reanimated` shared values
- Track `baseScale` (starts at computed scale) and `pinchScale` (gesture multiplier)
- Effective scale = baseScale * pinchScale, clamped between `computedScale * 0.5` and `2.0`
- On pinch end, update baseScale to current effective scale
- Use `useAnimatedStyle` to apply the animated scale transform
- The Gesture.Pinch handler updates the reanimated shared value, which drives the transform

**Important (Pitfall 1 from research):** Touch targets scale with the canvas. The pinch-to-zoom mitigates this -- users can zoom in to interact with small controls on phone screens. Additionally, the form_layout_config may have phone-specific layouts with larger controls.

**1. Create FormElementRenderer** (`apps/mobile/src/components/form/FormElementRenderer.tsx`):

Dispatcher component that maps `FormElementSpec.type` to the appropriate renderer.

Props:
```typescript
interface FormElementRendererProps {
  element: FormElementSpec;
  formData: Record<string, string>;
  onFormDataChange: (key: string, value: string) => void;
  images?: Map<string, string>;  // filename -> base64 URI
}
```

Switch on `element.type`:
- `'text'` -> `<TextElement>`
- `'header'` -> `<HeaderElement>`
- `'input'` -> `<InputElement>`
- `'image'` -> `<ImageElement>`
- `'video'` -> fallback (video not supported v1)
- `'checkbox'` -> `<CheckboxElement>`
- `'button'` -> `<ButtonElement>`
- `'select'` or `'dropdown'` -> `<SelectElement>`
- `'date'` or `'datepicker'` -> `<DatePickerElement>`
- `'textarea'` -> `<TextAreaElement>`
- `'number'` or `'numeric'` -> `<NumericInputElement>`
- `'toggle'` or `'switch'` -> `<ToggleSwitchElement>`
- `'radio'` or `'radiobutton'` -> `<RadioButtonElement>`
- default -> `<FallbackElement>` (gray box with element type text, never crashes)

All element type strings should be compared case-insensitively (`element.type.toLowerCase()`).

Each element renderer shares a common interface:
```typescript
interface ElementProps {
  element: FormElementSpec;
  value?: string;
  onChange?: (value: string) => void;
  images?: Map<string, string>;
}
```

The `value` is derived from `formData[element.content?.plainText || '']` (the plainText content acts as the field key). The `onChange` calls `onFormDataChange(key, newValue)`.

**2. Create element renderers:**

Each element is a small focused component:

**TextElement**: Renders `element.content?.content` as text. If content contains HTML-like formatting (the content field may have rich text from BrainPal MD), render as plain text (strip tags) for v1. Apply `fontSize: element.fontSize`, `color: element.color`, `textAlign: element.align`. Wrap in a View with the element's width/height.

**HeaderElement**: Same as TextElement but with bold fontWeight and larger default fontSize. Use `typography.heading` as base style, override with element.fontSize if provided.

**InputElement**: `<TextInput>` with border, placeholder from element.content?.plainText. Value from formData, onChange writes to formData. Style: white background, 1px border, border-radius 4, padding. Apply element dimensions.

**ImageElement**: Displays an image from the step's stored images. `element.src` contains the image filename. Look up in `images` Map to get base64 data URI. Render with `<Image source={{ uri: dataUri }}>` from react-native. If image not found, show placeholder box with image icon. Apply element width/height as image dimensions with `resizeMode: 'contain'`.

**CheckboxElement**: Pressable row with a checkbox indicator (square with/without checkmark) and label from `element.content?.plainText`. Toggles between "true"/"false" string values in formData.

**ButtonElement**: `<Pressable>` styled as a button with `element.content?.plainText` as label. Background color from `element.color` or primary color. This is for form-embedded buttons (not the main action buttons). OnPress triggers onChange with the button's plainText value.

**SelectElement**: For v1, render as a native Picker or a Pressable that opens a modal with options. Since the FormElementSpec doesn't specify options directly, this may need to parse from content or be data-driven. For v1, render a TextInput with a dropdown icon as a visual placeholder. If the element has options data in content, parse and render as a simple list picker.

**DatePickerElement**: Render a TextInput that shows the current date value. For v1, use a basic text input with date formatting hint. Full native date picker integration (expo-date-picker) is a nice-to-have -- for v1 a text input labeled "Date (YYYY-MM-DD)" is acceptable.

**TextAreaElement**: Multi-line `<TextInput>` with `multiline={true}` and `numberOfLines={4}`. Same styling pattern as InputElement but taller.

**NumericInputElement**: `<TextInput>` with `keyboardType="numeric"`. Same as InputElement but restricts to numeric input.

**ToggleSwitchElement**: React Native `<Switch>` component with label from content. Toggles between "true"/"false" in formData.

**RadioButtonElement**: A group of radio button options where only one can be selected at a time. Renders a vertical list of Pressable rows, each with a circular radio indicator (filled circle when selected, empty circle when not) and a label. Options come from `element.options` array (each option has a `label` and `value`). The selected value is stored in formData as a single string. On press, set formData to the option's value (deselecting any previously selected option). This is required by the CONTEXT.md locked decision for the extended control set ("radio buttons").

**FallbackElement** (inline in FormElementRenderer, not a separate file): Gray dashed-border box with text showing `Unsupported: ${element.type}`. Never crashes -- this is the safety net per Pitfall 6 from research.
