<task type="auto">
  <name>Task 1: Create useHistory hooks with SQLite queries</name>
  <files>apps/mobile/src/hooks/useHistory.ts</files>
  <action>
    Create `apps/mobile/src/hooks/useHistory.ts` with three hooks:

    **1. `useCompletedWorkflows()`**:
    - Uses `useSQLiteContext()` from expo-sqlite
    - Uses React `useState` + `useEffect` + `useCallback` for manual refresh
    - Query: `SELECT instance_id, master_workflow_oid, workflow_state, specification_json, created_at, started_at, completed_at FROM runtime_workflows WHERE workflow_state IN ('COMPLETED', 'ABORTED', 'STOPPED') AND parent_workflow_instance_id IS NULL ORDER BY completed_at DESC`
    - Parse `specification_json` to extract workflow name (`local_id` or `description`)
    - Return `{ workflows: HistoryWorkflow[], loading: boolean, refresh: () => void }`
    - Type `HistoryWorkflow` = `{ instanceId, name, state, startedAt, completedAt, duration }`
    - Compute duration as `completedAt - startedAt` in milliseconds, format as "Xm Ys" or "Xh Ym"

    **2. `useWorkflowHistory(instanceId: string)`**:
    - Query steps: `SELECT s.* FROM runtime_steps s JOIN runtime_workflows w ON s.workflow_instance_id = w.instance_id WHERE (w.instance_id = ? OR w.parent_workflow_instance_id = ?) AND s.step_type NOT IN ('START', 'END', 'PARALLEL', 'WAIT_ALL', 'WAIT_ANY') ORDER BY s.activated_at ASC` with `[instanceId, instanceId]`
    - For each step, parse `step_json` to get name (`local_id` or `description`), compute duration from `activated_at` to `completed_at`
    - Query audit log: `SELECT * FROM execution_log_entries WHERE workflow_instance_id = ? OR workflow_instance_id IN (SELECT instance_id FROM runtime_workflows WHERE parent_workflow_instance_id = ?) ORDER BY timestamp ASC` with `[instanceId, instanceId]`
    - Return `{ steps: HistoryStep[], logEntries: HistoryLogEntry[], loading: boolean }`
    - Type `HistoryStep` = `{ instanceId, name, stepType, state, duration, resolvedInputs, resolvedOutputs, userInputs }`
    - Type `HistoryLogEntry` = `{ id, stepOid, eventType, eventData, timestamp }`

    **3. `useDeleteWorkflow()`**:
    - Returns `async (instanceId: string) => Promise<void>`
    - Must delete execution_log_entries first (no ON DELETE CASCADE per Pitfall 6 from research): `DELETE FROM execution_log_entries WHERE workflow_instance_id = ? OR workflow_instance_id IN (SELECT instance_id FROM runtime_workflows WHERE parent_workflow_instance_id = ?)`
    - Then delete child workflows: `DELETE FROM runtime_workflows WHERE parent_workflow_instance_id = ?`
    - Then delete the parent workflow: `DELETE FROM runtime_workflows WHERE instance_id = ?`
    - Child runtime_steps and runtime_connections are cleaned up by ON DELETE CASCADE from runtime_workflows

    **Duration formatting helper** (private to module):
    - `formatDuration(ms: number): string` -- returns "< 1s" for < 1000ms, "Xs" for < 60s, "Xm Ys" for < 1h, "Xh Ym" for >= 1h
  </action>

  </verify>
