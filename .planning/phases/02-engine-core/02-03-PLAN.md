    **Create parameter resolver types** (`src/parameter-resolver/types.ts`):
    - `ResolvedParameter`: { id: string, value: string, source: 'literal' | 'property', property_name?: string, entry_name?: string }
    - `ParameterResolutionResult`: { resolved: ResolvedParameter[], errors: string[] }

    **Create ScopeResolver** (`src/parameter-resolver/scope-resolver.ts`):
    - Constructor takes IValuePropertyRepository and IWorkflowRepository
    - `lookupProperty(workflowInstanceId: string, propertyName: string, entryName: string)`: Promise<string | null>
      - Step 1: Look in workflow scope (workflow_value_properties for this workflowInstanceId)
      - Step 2: If not found and workflow has parent_workflow_instance_id, look in parent workflow scope (traverse up the chain)
      - Step 3: If not found in any workflow scope, look in environment scope (environment_value_properties)
      - Return first non-null value found, or null if not found at any scope
    - This scope chain is CRITICAL for correctness -- see Research Pitfall 5

    **Create ParameterResolver** (`src/parameter-resolver/parameter-resolver.ts`):
    - Constructor takes ScopeResolver
    - `resolveInputs(workflowInstanceId: string, parameters: ParameterSpecification[])`: Promise<ParameterResolutionResult>
      - For each parameter:
        - If value_type === 'literal': use default_value directly
        - If value_type === 'property': parse default_value to extract property_name and entry_name, then call scopeResolver.lookupProperty()
        - If lookup returns null: use default_value as fallback (do not error)
      - Return all resolved parameters
    - `writeOutputs(workflowInstanceId: string, outputs: OutputParameterSpecification[], resolvedValues: Map<string, string>)`: Promise<void>
      - For each output parameter:
        - Get the resolved value from resolvedValues map (keyed by parameter id)
        - Write to Value Property via IValuePropertyRepository.upsertEntry() using target_property_name and target_entry_name
        - Scope is always 'workflow' for workflow-scoped outputs, determined by output spec

    **Create condition evaluator types** (`src/condition-evaluator/types.ts`):
    - `Select1Config`: { inputValue: string, conditions: ConditionConnection[] }
    - `ConditionResult`: { matchedConnectionId: string } | null
    - `ConditionNotMatchedError` class with step info and input value

    **Create ConditionEvaluator** (`src/condition-evaluator/condition-evaluator.ts`):
    - `evaluateCondition(inputValue, operator, expectedValue, valueType)`: boolean
      - Coerce types based on valueType (number/integer/float -> Number(), boolean -> lowercase === 'true', default string)
      - Apply operator: equals, not_equals, greater_than, less_than, greater_than_or_equal, less_than_or_equal, contains, not_contains, starts_with, ends_with
      - Follow exact pattern from research Code Examples
    - `evaluate(config: Select1Config)`: string (returns matched connection_id)
      - Iterate conditions in order
      - Return first matching connection_id
      - If none match: throw ConditionNotMatchedError with inputValue and step info
      - This handles Pitfall 6 (SELECT 1 no-match case)

    **Create resource manager types** (`src/resource-manager/types.ts`):
    - `ResourcePool`: { id: string, resource_name: string, scope: 'workflow' | 'environment', scope_id: string, resource_type: ResourceType, capacity: number, current_usage: number, named_instances?: { name: string, acquired_by: string | null }[] }
    - `ResourceRequest`: { step_instance_id: string, workflow_instance_id: string, command_type: ResourceCommandType, resource_name: string, amount: number }
    - `AcquireResult`: { acquired: boolean, waitQueued: boolean }
    - `SyncBarrierEntry`: { resource_name: string, step_instance_id: string, workflow_instance_id: string, command_type: 'Synchronize' | 'Send' | 'Receive' }

    **Create ResourceManager** (`src/resource-manager/resource-manager.ts`):
    - Constructor takes IResourcePoolRepository, IResourceQueueRepository, ISyncBarrierRepository

    **Resource acquisition/release:**
    - `acquire(request: ResourceRequest)`: Promise<AcquireResult>
      - Find the resource pool by name + scope
      - Check if capacity allows (current_usage + amount <= capacity)
      - If available: increment usage, return { acquired: true }
      - If not available: add to FIFO queue, return { acquired: false, waitQueued: true }
      - **Deadlock prevention:** When a step acquires multiple resources, acquire them in ALPHABETICAL order by resource name. The ResourceManager receives all resource commands for a step and sorts them before processing.
    - `release(stepInstanceId: string, resourceName: string)`: Promise<void>
      - Decrement usage in pool
      - Check FIFO queue for next waiting request
      - If queue has a waiting step and capacity now allows: dequeue and grant (return step_instance_id of newly granted step so runner can resume it)
    - `releaseAll(workflowInstanceId: string)`: Promise<void>
      - Release all resources held by any step in the workflow

    **SYNC barriers:**
    - `registerSync(entry: SyncBarrierEntry)`: Promise<{ matched: boolean, partnerId?: string }>
      - For 'Synchronize': Look for another unmatched Synchronize on same resource_name. If found: match both, return matched=true with partner. If not found: register and wait.
      - For 'Send': Look for unmatched 'Receive' on same resource_name. If found: match. If not: register and wait.
      - For 'Receive': Look for unmatched 'Send' on same resource_name. If found: match. If not: register and wait.

    **Pool initialization:**
    - `initializePools(scope: 'workflow' | 'environment', scopeId: string, specs: ResourcePropertySpecification[])`: Promise<void>
      - Create resource pools from specifications
    - `destroyPools(scope: 'workflow' | 'environment', scopeId: string)`: Promise<void>
      - Clean up all pools for scope

    **Create barrel export** (`src/resource-manager/index.ts`).

    **Create resource manager tests** (`__tests__/resource-manager/resource-manager.test.ts`):
    Add InMemoryResourcePoolRepository, InMemoryResourceQueueRepository, InMemorySyncBarrierRepository to mock-repositories.ts (or create inline for these tests if not already defined).

    - Test binary exclusive: acquire succeeds first time, second acquire queues

    - Test Synchronize barrier: first step registers, second step matches both
    - Test Send/Receive: Send registers, Receive matches (and vice versa)
    - Test initializePools creates pools from specs
    - Test releaseAll releases all resources for a workflow
