  <action>
    **Step 1: Create the environment value property store.**

    Create `apps/mobile/src/stores/environment-store.ts`:

    - Define an `EnvironmentValueProperty` interface:
      ```typescript
      interface EnvironmentValueProperty {
        environment_oid: string;
        property_name: string;
        entries: Array<{ name: string; value: string }>;
        last_modified: string;
      }
      ```

    - Define an `EnvironmentStore` interface with:
      - `properties: EnvironmentValueProperty[]` -- the in-memory state
      - `isLoaded: boolean` -- whether store has been initialized from SQLite
      - `loadFromDb: (db: SQLiteDatabase) => Promise<void>` -- reads ALL rows from `environment_value_properties` and populates the store. Parses `entries_json` from string to parsed object array. Sets `isLoaded: true`.
      - `setProperty: (queue: WriteQueue, envOid: string, propName: string, entries: Array<{name: string; value: string}>) => Promise<void>` -- uses `WriteQueue.execute()` combined with `writeAhead` from `@brainpal/storage` to:
        1. `queue.execute(async (db) => { ... })` wraps the SQL operation for proper serialization (prevents web interleaving)
        2. Inside the queue callback, uses `writeAhead` to INSERT OR REPLACE into `environment_value_properties` in SQLite FIRST
        3. Only after SQL succeeds, updates Zustand state via the writeAhead stateUpdate callback
      - `getProperty: (envOid: string, propName: string) => EnvironmentValueProperty | undefined` -- pure in-memory lookup

    - Use `create<EnvironmentStore>()((set, get) => ({ ... }))` (Zustand v5 pattern with double parentheses).

    - Import `writeAhead` and `WriteQueue` from `@brainpal/storage`.
    - Import `type SQLiteDatabase` from `expo-sqlite`.

    **CRITICAL:** The `setProperty` method MUST use `WriteQueue.execute()` to serialize the write AND `writeAhead` inside it to enforce SQLite-before-Zustand ordering. Do NOT call `set()` before the SQLite write completes. Do NOT bypass the WriteQueue by calling `db.runAsync()` directly -- this prevents concurrent web writes from interleaving.

    **Step 3: Create the StoreInitializer provider.**

    Create `apps/mobile/src/providers/StoreInitializer.tsx`:

    ```tsx
    import { useSQLiteContext } from 'expo-sqlite';
    import { useEffect, useState } from 'react';
    import { View, ActivityIndicator, Text } from 'react-native';
    import { useEnvironmentStore } from '../stores/environment-store';
    import { useWorkflowStore } from '../stores/workflow-store';
    import { colors } from '@brainpal/ui';

    export function StoreInitializer({ children }: { children: React.ReactNode }) {
      const db = useSQLiteContext();
      const [ready, setReady] = useState(false);
      const [error, setError] = useState<string | null>(null);
      const loadEnvironment = useEnvironmentStore((s) => s.loadFromDb);
      const loadWorkflows = useWorkflowStore((s) => s.loadFromDb);

      useEffect(() => {
        async function initStores() {
          try {
            await loadEnvironment(db);
            await loadWorkflows(db);
            setReady(true);
          } catch (err) {
            console.error('Store initialization failed:', err);
            setError(err instanceof Error ? err.message : 'Unknown error');
          }
        }
        initStores();
      }, [db]);

      if (error) {
        return (
          <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center', padding: 16 }}>
            <Text style={{ color: colors.error, fontSize: 16 }}>Store initialization failed</Text>
            <Text style={{ color: colors.textSecondary, marginTop: 8 }}>{error}</Text>
          </View>
        );
      }

      if (!ready) {
        return (
          <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center', backgroundColor: colors.background }}>
            <ActivityIndicator size="large" color={colors.primary} />
          </View>
        );
      }

      return <>{children}</>;
    }
    ```

    **Step 4: Wire StoreInitializer into the root layout.**

    Update `apps/mobile/app/_layout.tsx` to wrap content in StoreInitializer INSIDE SQLiteProvider:

    ```tsx
    import { Stack } from 'expo-router';
    import { SQLiteProvider } from 'expo-sqlite';
    import { initializeDatabase } from '@brainpal/storage';
    import { StoreInitializer } from '../src/providers/StoreInitializer';

    export default function RootLayout() {
      return (
        <SQLiteProvider
          databaseName="brainpal_mobile.db"
          onInit={initializeDatabase}
        >
          <StoreInitializer>
            <Stack>
              <Stack.Screen name="(tabs)" options={{ headerShown: false }} />
            </Stack>
          </StoreInitializer>
        </SQLiteProvider>
      );
    }
    ```

    The nesting order is critical: `SQLiteProvider` (provides db) -> `StoreInitializer` (uses db to load stores) -> `Stack` (renders tabs with populated stores).

    **ANTI-PATTERNS TO AVOID:**
    - Do NOT use Zustand's `persist` middleware. SQLite is the single source of truth. Zustand is ephemeral.
    - Do NOT initialize Zustand stores with hardcoded data. Always load from SQLite.
    - Do NOT render meaningful UI before StoreInitializer sets `ready = true`.
