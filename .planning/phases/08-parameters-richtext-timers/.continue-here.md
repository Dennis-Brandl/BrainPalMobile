<decisions_made>

- Timer auto-starts on mount (RUNNING state) — matches BrainPal MD UX where timers start when the step activates
- blockDone mechanism: timers report blocking state via callback chain (TimerElement → FormCanvas → StepCarousel), FormCanvas disables all buttons when any timer is blocking
- Parameter chip regex: `<span data-param-chip="name.key">...</span>` — matches BrainPal MD's editor output format
- Schema migration: bumped user_version from 1 to 2, v1→v2 migration just flips the STEP_ATTENTION default
- resolveParameter/writeFormOutputParameters are public WorkflowRunner methods (not on ParameterResolver) so the UI can call them directly without knowing about scope resolution internals
</decisions_made>
